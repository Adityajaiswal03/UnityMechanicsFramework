PURPOSE:
A demo-only swing animator. Listens for WeaponFiredEvent and rotates a
target Transform through an arc to visualise a melee strike. Stays
fully decoupled from MeleeWeapon_UMFOSS via the event bus — drop it on
any GameObject, point it at a sword sprite, and it will animate
whenever a matching melee weapon fires. An optional weapon-name filter
lets a single scene hold multiple melee weapons without them all
swinging together.

Code    : if (swordVisual != null) { swordVisual.localRotation = Quaternion.Euler(0f, 0f, startAngle); }

Code    : private void HandleWeaponFired(WeaponFiredEvent evt)
Explain : The fired-event handler. Runs for every weapon fire the

Code    : if (swordVisual == null) { return; }
Explain : Fail gracefully if no visual is wired. The event still
          dispatched to other subscribers — this one just does
          nothing.

Code    : if (!string.IsNullOrEmpty(weaponNameFilter) && evt.data.name != weaponNameFilter) { return; }
Explain : Applies the optional name filter. When the filter is
          empty, every fire event triggers the swing; when set,
          only events whose weapon name matches trigger the swing.

Code    : if (activeSwing != null) { StopCoroutine(activeSwing); }
Explain : Cancels any swing still in progress so a rapid re-fire
          restarts the animation from the top instead of stacking
          two coroutines on the same transform.

Code    : activeSwing = StartCoroutine(SwingRoutine());
Explain : Launches the swing routine and stores the handle.

Code    : private IEnumerator SwingRoutine()
Explain : The private coroutine that drives the rotation each frame.

Code    : float elapsed = 0f;
Explain : Local timer used to compute the 0..1 animation parameter.

Code    : while (elapsed < swingDuration)
Explain : Runs the animation loop until the swing duration elapses.

Code    : float t = elapsed / swingDuration;
Explain : Normalized time across the swing, 0 at start, 1 at end.

Code    : float eased = 1f - (1f - t) * (1f - t);
Explain : Quadratic ease-out curve. At t=0 the output is 0, at t=1
          it is 1, and the curve moves fast early and slows down —
          which gives the swing a "snap forward then settle" feel
          rather than a mechanical linear arc.

Code    : float angle = Mathf.Lerp(startAngle, endAngle, eased);
Explain : Lerps the rotation angle between start and end using the
          eased parameter, producing the curved feel.

Code    : swordVisual.localRotation = Quaternion.Euler(0f, 0f, angle);
Explain : Applies the rotation as a pure Z-axis quaternion, which is
          all that matters for a 2D sword sprite.

Code    : elapsed += Time.deltaTime;
Explain : Advances the local timer using scaled game time so
          slow-motion correctly extends the animation.

Code    : yield return null;
Explain : Waits one frame before continuing.

Code    : swordVisual.localRotation = Quaternion.Euler(0f, 0f, startAngle);
Explain : Snaps back to the start pose once the arc completes, so
          the next swing has somewhere to swing from.

Code    : activeSwing = null;
Explain : Clears the handle so HandleWeaponFired can tell the
          coroutine has finished naturally.

DESIGN NOTES:

- Why listen through the event bus instead of referencing the melee
  weapon?
  Decoupling. The animator has zero knowledge of which weapon
  instance fired. Drop it in any scene, wire a sword sprite, and it
  works without editing the animator to point at the weapon
  component.

- Why an ease-out curve instead of linear Lerp?
  Linear motion looks lifeless on combat animations. The quadratic
  ease-out snaps forward at the start and settles at the end,
  producing a recognisable "strike" feel with zero assets or
  Animator state machines.

- Why a name filter?
  A scene could have multiple melee weapons (player + enemy). Without
  the filter, the player's sword would swing whenever the enemy
  attacks. The filter lets each animator pick exactly which weapon's
  events it should react to.
